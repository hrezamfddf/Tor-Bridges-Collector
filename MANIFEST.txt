BUG FIX 1: dynamic_model_brain.py — CF model fetch returns 0 models
  ROOT CAUSE: URL used ?task=text-generation&per_page=100 which CF API
              ignores (wrong task identifier). Returns 0 models always.
  FIX:       Uses ?per_page=500 (same as model_selector.py which works
              and returns 25 models). Filters by text-gen task in Python.
              Also retries with ?per_page=500 (no search) as fallback.
  RESULT:    Brain fetches 25+ live models from first CF account.

BUG FIX 2: dynamic_model_brain.py — Brain stops on first account
  ROOT CAUSE: refresh() broke out of loop on ANY successful account,
              even if it returned 0 models (due to Bug 1 or 403).
  FIX:       Only breaks when models are actually returned. Logs
              progress when skipping empty accounts. Falls back to
              offline model list when ALL accounts return 0 models.

BUG FIX 3: providers.py — CF AI Gateway HTTP 400 on all 11 slots
  ROOT CAUSE: CF AI Gateway OpenAI-compat endpoint may reject requests
              depending on gateway version/config. All 6 model IDs
              with @cf/ prefix failed on all 11 gateway slugs.
  FIX A:     Also tries model names WITHOUT @cf/ prefix (bare format).
  FIX B:     After all /v1/chat/completions variants fail, retries with
              native CF Workers AI URL format:
              {gateway_base}/workers-ai/@cf/{model_id}
  FIX C:     Logs actual 400 response body (previously swallowed).
  FIX D:     Supports CF_AI_GATEWAY_TOKEN_{n} env var for gateway-
              specific cf-aig-authorization header (some gateways require it).
  FIX E:     ProviderConfigurationError with diagnostic checklist when
              all slots/models fail — guides user to fix the root cause.

BUG FIX 4: providers.py — Portkey HTTP 400 on all 3 slots
  ROOT CAUSE: Portkey API without x-portkey-provider header returns 400
              because it doesn't know which backend to route to.
              PORTKEY_PROVIDER_KEY was empty, so no routing header was added.
  FIX:       Always adds x-portkey-provider: cerebras for Cerebras models.
              Uses CEREBRAS_API_KEY_1 as backend auth when available.
              Adds x-portkey-custom-host: https://api.cerebras.ai so
              Portkey proxies to Cerebras directly.
              Tries llama3.1-8b first (most reliable Cerebras model).

BUG FIX 5: .github/workflows/ai_gateway_health_check.yml
  ROOT CAUSE: Pre-flight step only exposed CF_ACCOUNT_ID/TOKEN 1-3 but
              health check uses all 11 slots. Slots 4-11 had no env vars
              visible to the pre-flight validation script.
  FIX:       Pre-flight now exposes all 11 CF_ACCOUNT_ID, CF_API_TOKEN,
              and CF_AI_GATEWAY_URL variables (1-11).
              Health check default task changed to 'fast' (was 'general').
              Health check version echo corrected to v13.0.
              CEREBRAS_API_KEY_1 passed to health check for Portkey fix.
